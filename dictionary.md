# Словарь ключевых терминов в алгоритмах


---

## Анализ сложности

**Big-O (асимптотическая сложность)** — оценка роста числа операций при увеличении размера входа `n`. Записывается `O(f(n))`. Это верхняя граница: «не хуже чем».

**Big-Ω (омега)** — нижняя граница, «не лучше чем». Редко используется на практике.

**Big-Θ (тета)** — точная граница (и сверху, и снизу одновременно).

**Амортизированная сложность** — средняя стоимость операции на длинной последовательности операций. Классика: добавление в динамический массив. Одна операция может быть `O(n)` при расширении, но амортизированно — `O(1)`, потому что расширения редки.

**Время vs память** — две разные сложности. Часто есть компромисс: можно ускориться, потратив больше памяти (мемоизация), или наоборот — сэкономить память, пересчитывая.

**Worst case / Average case / Best case** — худший, средний, лучший случай. Quicksort: `O(n log n)` средний, `O(n²)` худший.

**TLE (Time Limit Exceeded)** — ответ правильный, но слишком медленно. Обычно значит, что ты выбрал не тот алгоритм под размер входа.

**MLE (Memory Limit Exceeded)** — алгоритм съел слишком много памяти.

## Структуры данных: базовые понятия

**Коллизия** — ситуация в хеш-таблице, когда два разных ключа дают одинаковый хеш. Разрешается либо **цепочками** (chaining — в ячейке лежит список всех элементов с этим хешем), либо **открытой адресацией** (open addressing — ищем следующую свободную ячейку).

**Хеш-функция** — функция, превращающая ключ в целое число-индекс. Хорошая хеш-функция распределяет ключи равномерно.

**Load factor (коэффициент заполнения)** — отношение числа элементов к размеру хеш-таблицы. Когда превышает порог (обычно 0.75), таблица **ресайзится** — создаётся большая, все элементы перехешируются.

**Указатель / ссылка** — адрес элемента в памяти. `null` / `None` / `nullptr` — «указатель в никуда».

**LIFO (Last In, First Out)** — последний вошёл, первый вышел. Это стек.

**FIFO (First In, First Out)** — первый вошёл, первый вышел. Это очередь.

**Куча (heap)** — дерево, где родитель всегда ≤ (min-heap) или ≥ (max-heap) детей. Даёт быстрый доступ к минимуму/максимуму.

**Приоритетная очередь (priority queue)** — абстракция, обычно реализованная через кучу. Извлекает элементы не в порядке добавления, а в порядке приоритета.

**Буфер / массив фиксированного размера** — заранее выделенная область памяти.

**Слайс / подмассив (subarray)** — **непрерывный** участок массива. Важно: subarray ≠ subsequence.

**Подпоследовательность (subsequence)** — элементы в исходном порядке, но **не обязательно подряд**. `[1,3,5]` — подпоследовательность `[1,2,3,4,5]`, но не подмассив.

## Деревья и графы

**Узел (node, vertex)** — точка в графе или дереве.

**Ребро (edge)** — связь между двумя узлами. Бывает **направленным** (ориентированным) или нет.

**Степень узла (degree)** — сколько рёбер из него выходит. В ориентированном графе — **in-degree** и **out-degree** отдельно.

**Путь (path)** — последовательность узлов, где каждые два соседних соединены ребром.

**Цикл** — путь, возвращающийся в стартовую точку. Граф без циклов называется **ациклическим**. **DAG** = Directed Acyclic Graph — направленный ациклический граф (часто встречается: зависимости задач, выражения, генеалогия).

**Связный граф** — из любого узла можно дойти до любого другого.

**Компонента связности** — максимальное подмножество узлов, между которыми есть пути.

**Взвешенный граф** — у рёбер есть веса (стоимость, расстояние). Дейкстра работает именно с такими.

**Плотный / разрежённый граф** — много рёбер (`E ≈ V²`) vs мало (`E ≈ V`). От этого зависит выбор между матрицей смежности и списком смежности.

**Матрица смежности** — квадратная матрица `n×n`, где `M[i][j] = 1`, если есть ребро. Память `O(V²)`.

**Список смежности** — для каждого узла хранится список его соседей. Память `O(V + E)`. В большинстве задач — используется он.

**Корень (root)** — верхний узел дерева. У дерева один корень.

**Лист (leaf)** — узел без детей.

**Предок / потомок (ancestor / descendant)** — узлы выше/ниже по дереву.

**LCA (Lowest Common Ancestor)** — ближайший общий предок двух узлов.

**Высота (height)** — длина самого длинного пути от узла до листа. Высота дерева = высота корня.

**Глубина (depth)** — расстояние от корня до узла. Корень на глубине 0.

**BST (Binary Search Tree)** — дерево поиска: для каждого узла всё в левом поддереве меньше, в правом — больше.

**Сбалансированное дерево** — высота `O(log n)`. AVL, красно-чёрное, B-дерево — примеры. TreeMap в Java / std::map в C++ — обычно красно-чёрные.

**Обход дерева:**
- **Pre-order**: корень → левое → правое
- **In-order**: левое → корень → правое (на BST даёт отсортированный порядок)
- **Post-order**: левое → правое → корень
- **Level-order (BFS)**: по уровням сверху вниз

## Рекурсия

**Рекурсия** — функция, вызывающая сама себя на меньшей задаче.

**База рекурсии (base case)** — условие, при котором функция возвращает ответ без рекурсивного вызова. Без базы = бесконечная рекурсия = **stack overflow**.

**Рекурсивный шаг** — как свести задачу к одной или нескольким меньшим.

**Call stack (стек вызовов)** — где живут активные вызовы функции. Каждый рекурсивный вызов добавляет фрейм; возврат — снимает.

**Глубина рекурсии** — максимальное число вложенных вызовов. Если больше ~1000 (Python) или ~10⁴ (C++) — переполняется стек.

**Хвостовая рекурсия (tail recursion)** — когда рекурсивный вызов — последнее действие функции. Некоторые языки умеют её оптимизировать в цикл. Python, Java — не умеют.

**Мемоизация** — кеширование результатов рекурсивной функции, чтобы не пересчитывать одно и то же. Обычно через словарь или декоратор `@lru_cache`.

## Техники и парадигмы

**Brute force (полный перебор)** — перебираем все варианты. Медленно, но всегда правильно. Хорошо как первая версия решения, от которой отталкиваешься к оптимальному.

**Divide and conquer (разделяй и властвуй)** — разбиваем задачу на независимые подзадачи, решаем рекурсивно, объединяем ответы. Merge sort, quick sort, быстрое возведение в степень.

**Dynamic Programming (DP, динамическое программирование)** — решаем перекрывающиеся подзадачи и кешируем. Две формы:
- **Top-down** — рекурсия + мемоизация
- **Bottom-up** — итеративно заполняем таблицу

**Перекрывающиеся подзадачи** — наивная рекурсия решает одну и ту же подзадачу много раз. Признак, что нужна DP.

**Оптимальная подструктура** — оптимальный ответ задачи строится из оптимальных ответов подзадач. Без этого свойства DP не работает.

**Greedy (жадный алгоритм)** — на каждом шаге делаем локально оптимальный выбор, не пересматривая прошлые. Работает не всегда — нужно доказывать.

**Backtracking (откат)** — перебор с возможностью отмены выбора. Строим решение инкрементально; если зашли в тупик — откатываемся и пробуем другую ветвь.

**Pruning (отсечение)** — в backtracking и переборе: пропускаем ветви, про которые уже понятно, что они не приведут к ответу. Сильно ускоряет.

**Two pointers** — два индекса, движущиеся по массиву (навстречу, параллельно или с разными скоростями).

**Sliding window (скользящее окно)** — поддерживаем окно индексов `[l..r]`, расширяя и сужая. Каждый элемент входит и выходит не более одного раза → `O(n)`.

**In-place** — алгоритм, не требующий дополнительной памяти пропорционально входу. Работает «прямо в массиве». Обычно `O(1)` дополнительной памяти.

**Stable sort (устойчивая сортировка)** — элементы с одинаковыми ключами сохраняют относительный порядок. Merge sort — устойчив, quick sort — нет.

## Графовые алгоритмы

**BFS (Breadth-First Search, поиск в ширину)** — обход по уровням, с очередью. Даёт кратчайший путь в невзвешенном графе.

**DFS (Depth-First Search, поиск в глубину)** — обход «в глубину до упора», с рекурсией или стеком. Используется для поиска циклов, топологической сортировки, компонент связности.

**Топологическая сортировка** — упорядочивание узлов DAG так, что все рёбра идут слева направо. Работает только для DAG.

**Dijkstra (Дейкстра)** — кратчайшие пути в графе с **неотрицательными** весами.

**Bellman-Ford** — кратчайшие пути, работает и с отрицательными весами. Умеет находить отрицательные циклы. Медленнее Дейкстры.

**Floyd-Warshall** — кратчайшие пути **между всеми парами** узлов. `O(V³)`.

**MST (Minimum Spanning Tree, минимальное остовное дерево)** — подмножество рёбер, соединяющее все узлы с минимальной суммой весов. Алгоритмы: Kruskal, Prim.

**Union-Find (DSU, система непересекающихся множеств)** — структура для быстрой проверки связности и объединения компонент. Почти `O(1)` на операцию с оптимизациями.

**Visited set (множество посещённых)** — в BFS/DFS: куда мы уже заходили, чтобы не зациклиться.

## Битовые операции

**Бит** — одна двоичная цифра (0 или 1).

**AND (`&`)** — 1, если оба бита 1.

**OR (`|`)** — 1, если хотя бы один 1.

**XOR (`^`)** — 1, если биты разные. Ключевое свойство: `a ^ a = 0`, `a ^ 0 = a`.

**NOT (`~`)** — инверсия всех битов.

**Сдвиг влево (`<<`)** — умножение на 2. `x << k` = `x · 2^k`.

**Сдвиг вправо (`>>`)** — деление на 2 (целочисленное).

**Битовая маска** — число, где каждый бит кодирует присутствие/отсутствие элемента. `0b1011` = «есть элементы 0, 1, 3». Используется для перебора подмножеств при `n ≤ 20`.

**Дополнительный код (two's complement)** — способ представления отрицательных чисел. Отсюда трюк `x & -x` выделяет младший единичный бит.

## Практические термины

**Edge case (крайний случай)** — пустой вход, один элемент, все одинаковые, максимальный размер. То, на чём ломаются наивные решения.

**Off-by-one error** — ошибка «на единицу»: `<` вместо `<=`, `n` вместо `n-1`. Частая причина багов в циклах и бинарном поиске.

**Overflow (переполнение)** — значение не влезло в тип. Пример: `int32` переполняется на `2·10⁹`. В Python целые бесконечной точности, поэтому не встречается; в Java/C++ — постоянная проблема.

**Invariant (инвариант)** — утверждение, остающееся истинным на каждой итерации цикла или рекурсии. Инструмент доказательства корректности.

**Monotonic (монотонный)** — неубывающий или невозрастающий. Monotonic stack / deque — структура, где элементы лежат монотонно. Даёт `O(n)` в задачах вроде «следующий больший элемент».

**Sentinel (стражевой элемент)** — искусственный элемент на границе, упрощающий логику и убирающий проверки границ.

**Deduplication (дедупликация)** — убрать дубликаты. Обычно через `set`.

**Early exit / short-circuit** — выход из функции/цикла сразу, как найден ответ, без завершения всех итераций.

## Слова из условий задач

Часто в описании задач встречаются устоявшиеся термины:

**Distinct** — все различны (дубликатов нет).

**Non-decreasing** — неубывающий (`≤`, может повторяться). **Strictly increasing** — строго возрастающий (`<`).

**Contiguous** — непрерывный (subarray, а не subsequence).

**Lexicographic order** — лексикографический порядок (как в словаре). `"abc" < "abd" < "b"`.

**Palindrome** — палиндром: читается одинаково в обе стороны.

**Anagram** — анаграмма: одни и те же буквы в другом порядке.

**Permutation** — перестановка: те же элементы в другом порядке.

**Combination** — комбинация: подмножество фиксированного размера, порядок не важен.

**In-place** — «на месте», без дополнительной памяти.
